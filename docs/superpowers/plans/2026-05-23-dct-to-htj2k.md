# DCT To HTJ2K Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a validated coefficient-domain JPEG DCT coefficients to HTJ2K wavelet coefficients path, ending in an experimental JPEG-to-HTJ2K codestream transcode API.

**Architecture:** Keep cross-codec work in `signinum-transcode` so `signinum-jpeg` and `signinum-j2k` do not depend on each other. Prove the math in synthetic scalar code first, then wire codec extraction and HTJ2K packetization once the DCT-to-DWT stages have measurable correctness gates. Treat floating-point matrix composition as the linear oracle and integer 5/3 as a separate piecewise-integer proof or bounded-error implementation.

**Tech Stack:** Rust 1.94, Cargo workspace, `signinum-transcode`, `signinum-jpeg`, `signinum-j2k-native`, HTJ2K 5/3 reversible transform, TDD.

---

## File Structure

- `crates/signinum-transcode/src/lib.rs`: crate-level module exports.
- `crates/signinum-transcode/src/dct53_1d.rs`: 1D synthetic DCT-to-single-level-5/3 proof code.
- `crates/signinum-transcode/src/dct53_2d.rs`: 2D separable DCT-to-single-level-5/3 proof code.
- `crates/signinum-transcode/src/dct53_multilevel.rs`: LL-only follow-on decomposition after first-level output exists.
- `crates/signinum-transcode/src/jpeg_blocks.rs`: borrowed and owned DCT coefficient block descriptors for real JPEG integration.
- `crates/signinum-transcode/src/htj2k_wavelet.rs`: HTJ2K-ready wavelet band descriptors and validation helpers.
- `crates/signinum-transcode/src/error.rs`: explicit experimental error types.
- `crates/signinum-transcode/tests/dct53_1d.rs`: 1D synthetic tests.
- `crates/signinum-transcode/tests/dct53_2d.rs`: 2D synthetic tests.
- `crates/signinum-transcode/tests/dct53_multilevel.rs`: multilevel tests.
- `crates/signinum-transcode/tests/jpeg_blocks.rs`: JPEG block extraction tests.
- `crates/signinum-transcode/tests/htj2k_codestream.rs`: codestream integration tests.
- `crates/signinum-jpeg/src/transcode.rs`: public experimental DCT block extraction API when real JPEG integration begins.
- `crates/signinum-j2k-native/src/j2c/encode_coefficients.rs`: encoder path accepting precomputed wavelet bands.
- `docs/dct-to-htj2k-notes.md`: research notes, error histograms, and phase results.
